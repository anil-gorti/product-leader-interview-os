# Contributing

## What Makes a Good PR

### New skill files
If you have built a skill that works well for senior PM interviews —
a negotiation simulator, a product case builder, a metrics pressure test,
a market sizing skill — open a PR with your SKILL.md following
the format in the existing skills directory.

A strong skill file has:
- A specific trigger (the exact phrase that should activate it)
- A process (not just output format — the actual reasoning steps)
- Per-audience translation where the skill output differs by audience
- A warning section naming when NOT to use this skill

### India-specific context additions
The `knowledge/india-context/` directory is a living resource.
Additions welcome for:
- City-specific ecosystem dynamics (Pune, Chennai, Hyderabad are underrepresented)
- Specific sector contexts (edtech, healthtech, fintech, deeptech)
- Funding stage dynamics that are specific to India rounds
- Cultural translation notes from specific operating experiences

### Anonymised examples
Strong anonymised examples in `/examples/` help new users understand
the depth and specificity required. If you have filled these templates
for a real interview and can anonymise them fully, consider submitting.

## Format Requirements
- Markdown only
- No company names in public files
- No personal information
- Skill files must follow the existing SKILL.md format exactly

## What This Repo Is Not For
- Promoting specific tools or vendors
- Generic PM content that is not specific to senior leadership
- Content that is not applicable to the Indian product ecosystem
